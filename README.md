# 平行眼 VR 图库 / Parallel VR Gallery

本地 Android 平行眼 VR 图库。App 可以浏览系统图片和视频，在手机本地生成并缓存平行眼 SBS VR 图片；视频可作为播放器浏览，也可以手动加入 VR 生成队列。

## v2.9 修复

- 视频生成任务增加持久化标签索引：`queue_index/video_queue.tsv`。
- 视频加入队列、生成中、暂停、失败都会落盘记录，并保存当时的视频模型、CPU/GPU、深度强度、平滑、填充、最大长边等参数快照；生成成功或删除缓存时清理标签。
- App 被杀后重开，如果视频未生成完成，会恢复为“已暂停”而不是“未生成”。
- 重启后不会自动继续视频生成，需要在视频页或生成页手动点“继续生成”；继续时沿用上次任务参数，想用新参数则走“重新生成”。

## v2.8 修复

- 修复深度下滑后从图片预览返回定位不准：时间线会按保存的 LazyGrid 位置提前扩展渲染范围。
- 主图库视频缩略图状态改为读取视频 VR 状态，已生成视频不再显示为“原图”。
- 缩略图缩小时，状态标签和播放标记的字体、padding、遮罩同步缩小，减少小格子里遮罩偏大的问题。

## v2.7 修复

- 图片生成队列来源升级为持久化标签索引，保存 `ALL / ALBUM / GENERATED` 来源、相册 id、优先级和状态。
- App 启动时仍先加载系统图库，再异步恢复生成标签；已生成缓存不重复入队，未完成任务恢复为队列或暂停状态。
- 关闭 VR 会清理未完成队列标签，失败标签保留用于排查。
- 全部页分页追加后的日期分组改为本地日期 key，避免后续加载内容跨日期时 header 丢失或错位。
- 时间线渲染上限不再因加载更多而重置回首批数量，继续下滑时日期 header 更稳定。

## v2.6 修复

- 全部页、相册页、生成图片管理增加“顶部”按钮，滚动较深后可一键回到顶部。
- 生成页图片缩略图改为从已生成 SBS 文件左半边解码预览，避免已生成文件存在但管理页显示白屏。
- 图片预加载队列增加来源标记：全部页任务只在全部页继续，相册任务只在对应相册继续。
- 生成页是管理视图，不会把全部/相册来源任务改成生成页来源；只有在生成页点“重新生成”的任务才标记为生成页来源。
- 打开生成页时允许所有图片来源队列继续运行，方便统一管理。

## v2.5 优化

- SBS 合成改为按行低内存算法，不再同时持有左右眼整图和多份整图像素数组，降低大图 OOM 概率。
- SBS 合成内层循环预计算 X 到深度图索引，减少 CPU 重复计算；内存预算按新算法重新放宽，避免过度降分辨率。
- 首页双指缩小下限从 72dp 放宽到 42dp，竖屏可显示约 6 到 8 列。
- 右侧滚动条增加可拖动热区，可以按住侧边条快速上下跳转。

## v2.4 修复

- 从全部页或相册详情进入 Viewer 后，返回使用打开时记录的原始行位置，不再按 Viewer 当前页重新计算导致偏移。
- 相册详情页从 Viewer 返回时会用保存的滚动位置初始化，避免回到顶部或明显错位。
- 视频页只允许当前页播放；Pager 预加载到相邻视频时不会自动播放声音，离开页面会停止播放。
- 图片 VR 生成增加堆内存预算保护：内存不足时自动降低本次输出长边，并在 SBS 合成 OOM 时降一档重试，减少大图生成失败。

## v2.3 修复

- 普通“全部/相册”缩略图固定显示系统原图，不再在 VR READY 后替换成 SBS 输出，避免外部缩略图变黑或显示黑边。
- 底部“全部 / 相册 / 生成”切换保留各自位置；从相册详情切出去再切回，会回到原来的具体相册。
- 从相册详情打开图片后，返回会先回到该相册详情；在相册详情再返回才回到相册主页。

## v2.2 修复

- 全部页、相册列表、相册详情使用独立滚动状态，底部三个入口切换时不再共用同一个位置。
- 全部页、相册列表、相册详情增加右侧滚动条。
- 模型下载增加多个备用源：GitHub 原始地址失败后会自动尝试代理源；Qualcomm 实验模型增加 HuggingFace 镜像。
- 生成失败时如果是模型下载失败，错误会显示所有下载源的失败原因。

## v2.1 修复

- 全部页改为统一 `MediaStore.Files` 分页查询，图片和视频按同一时间线混排，接近底部时自动预取下一页。
- 相册详情继续按当前相册 `bucketId` 分页加载，接近底部自动加载下一页。
- 从全部页打开 Viewer 时，左右滑动和 VR 预加载按全部页当前加载范围执行。
- 从相册详情打开 Viewer 时，左右滑动和 VR 预加载只按当前相册范围执行，不再串到全局图库。
- 关闭 Viewer 后不再触发一次全局预加载，避免相册来源被清空后错误排队。

## v2.0 修复

- 相册详情改为按 `bucketId` 分页加载，点进相册先加载一页，继续下滑自动加载后续页，不再受首页最近 1800 项限制。
- 相册列表仍会后台全量聚合，数量保持完整；相册详情按页查询当前相册内容。
- 版本号提升到 `2.0`。

## v1.24 修复

- 修复全部页下滑几屏后可能闪退的问题：时间线日期 header key 改为稳定 epoch-day，避免跨年份同名日期导致 LazyGrid key 冲突。
- 图片和视频缩略图统一优先使用系统 `loadThumbnail`，避免下滑时大量完整图片解码造成内存压力。
- 缩略图占位去掉大量进度圈，降低列表滚动时的组合和绘制压力。
- 相册页启动时先显示最近媒体的临时聚合，后台再全量聚合系统相册并刷新，避免相册永远只有少数几个。

## v1.23 修复

- 修复 v1.21 首页改造后，大图库打开会卡住的问题：移除未选择状态下的全量 `indexOfFirst` 扫描，避免几万张媒体触发近似 N² 主线程计算。
- 首次打开只从 MediaStore 读取最近 1800 项媒体，避免启动阶段等待几万条记录；生成页仍可独立显示内部缓存。
- 全部/相册时间线改为渐进渲染，首屏先显示 1200 项，继续下滑自动加载更多，避免一次性把几万项交给 Compose。
- 生成页开始改为 App 内部生成库优先：图片/视频缓存即使不在本次系统相册扫描结果里，也会从 `params.json` / `video_params.txt` 恢复成生成页专用虚拟项。
- 内部生成库虚拟项不会出现在 `全部` 和 `相册` 页，避免污染普通图库浏览。

## v1.22 修复

- 修复几万张图片/视频图库加载时首页卡住的问题。
- 媒体加载阶段不再逐张图片/视频查找 VR 缓存，改为只扫描已有缓存目录后反查媒体项。
- UI 状态不再为每个普通未生成媒体创建 `NORMAL` 状态记录，只保留 READY/任务状态，减少大图库内存和主线程压力。
- `全部` 日期时间线改为线性生成 header/item，不再用 `groupBy` 复制大列表。
- 进入相册详情时不再复用全部页的大滚动位置，避免相册项较少时滚动越界。

## v1.21 功能

- 首页改为系统图库式结构：底部 `全部 / 相册 / 生成` 三个入口。
- `全部` 按日期分组显示系统图片和视频，视频带播放标记。
- `相册` 按系统 MediaStore 文件夹聚合，显示封面、名称和数量；进入相册后只浏览该相册内容。
- `生成` 直接替代旧“生成管理”，只显示已生成、生成中、暂停或失败的图片/视频 VR 缓存任务。
- Viewer 的左右滑动范围会跟随来源：从全部进入就浏览全部，从相册进入就浏览该相册，从生成进入就只浏览生成集合。
- 生成页仍支持图片/视频分页、多选、保存、重新生成和删除。

## 模型资产

APK 不内置 `depth_anything_v2.tflite`。首次生成 VR 前，App 会从 GitHub Release 下载模型并校验 SHA-256：

```text
https://github.com/7116-byte/ParallelVrGallery/releases/download/model-assets-v1/depth_anything_v2.tflite
```

已下载模型保存在 App 私有外部目录。覆盖安装升级通常不会清除该目录，因此本地已有模型时不需要重复下载；卸载 App 会由系统清理私有目录。

## 第三方项目

本项目使用 Android、Jetpack Compose、TensorFlow Lite、TFLite GPU Delegate，以及 Depth-Anything-V2 模型。SBS 生成思路参考了 `SamSeenX/ComfyUI_SSStereoscope` 中的深度平滑、相对视差和边缘填充做法。详细说明见 `NOTICE.md`。
