# 平行眼 VR 图库 / Parallel VR Gallery

本地 Android 平行眼 VR 图库。图片可本地生成并缓存 SBS VR 图；视频可作为播放器浏览，并可手动加入视频 VR 生成队列。

## v1.17 修复与改进

- 生成管理页支持长按多选，批量保存、重新生成、删除图片/视频 VR 缓存。
- 图片重新生成使用当前图片参数；视频重新生成使用当前视频参数。
- 暂停后继续仍沿用暂停时的原参数和原缓存，不会被设置页的新参数影响。
- 深度强度、深度平滑、边缘填充、输出最大长边移到公共设置区。
- 设置页移除顶部模型就绪状态，避免回到顶部反复看到“模型已就绪”；模型下载/校验只在生成前或切换模型后体现。
- 设置页调整为公共区、图片区、视频区的层级结构，减少标题拥挤。

## v1.16 修复与改进

- 设置页按“公共设置 / 图片生成设置 / 视频生成设置”分区，避免模型、图片、视频参数混在一起。
- 模型就绪状态改为快速检查，不再反复对大模型文件计算 SHA-256；生成前仍会校验模型哈希。
- 视频缓存按生成参数版本分目录，模型、CPU 线程、GPU 开关变化后会进入独立缓存。
- 暂停视频后继续生成会沿用暂停时的参数和缓存，不再清理已生成帧。
- 图片和视频缓存版本加入 CPU 线程数与 GPU 开关，修改这些设置后会真正重新生成。
- 图片生成日志记录 TFLite 线程数、GPU 请求和 delegate 是否启用，便于判断 CPU/GPU 是否生效。

## v1.15 修复与改进

- 图片模型和视频模型拆成独立选择项，图片生成与视频生成不再共用同一个模型设置。
- 旧版本的 `modelId` 会迁移为图片模型和视频模型的默认值，覆盖安装后行为保持一致。
- 设置页分别显示图片模型和视频模型状态，并在视频生成优化区域加入视频模型选择。
- 暂停中的视频继续生成时会清理该视频拆帧缓存，避免切换视频模型后混用旧帧。
- 视频参数文件记录实际使用的 `depthModel`，便于调试缓存来源。

## v1.14 修复与改进

- 图片 VR 页单点点击恢复为显示/隐藏信息控件切换。
- 视频“帧生成 xxms”改为最近三帧生成间隔的滚动平均，不再用开头前三帧。
- 视频排队或生成中时按钮改为“暂停生成”，暂停后可“继续生成”，便于修改参数做对比。
- 模型选择改为纵向按钮列表，长模型名不会挤成一团。
- 新增 Qualcomm Depth Anything V2 TFLite 实验模型下载入口。

## v1.13 修复与改进

- 视频进度条改为可拖动，拖动时直接 seek 到对应时间。
- 检查更新只有发现新版本时才显示“下载更新”按钮，已是最新版本时不再显示。
- 图片 VR 页“队列中”改为统计当前预加载窗口内前后各有多少张正在排队、生成或暂停，不再被中间已生成图片截断。
- VR 查看时左右半图分别贴向中线，避免因为居中适配在中线两侧留下黑边。

## v1.12 修复与改进

- 修复视频双击快进“识别了但原地跳转”的问题，改用底层播放器 seek。
- 视频页底部合并为一条控制区：播放/暂停、生成状态、时间和进度条放在一起，不再被额外信息层挡住。
- 视频进度条增加播放/暂停按钮。
- 检查更新结果提供“下载更新”按钮，会直接打开 GitHub APK 下载链接或 Release 页面。
- Android 13 及以上会补申请通知权限，视频生成通知才能正常出现在通知栏。
- 图片自动预加载最大只扩到前后 8，不再因为补位一路排到几百张之后。
- 视频帧拆分缓存保存到 App 私有持久目录，App 被杀后已生成帧不会被系统 cache 清理。
- 视频生成增加独立 CPU/GPU 设置，不再复用图片模型设置。
- 视频进度行增加前三帧平均生成时间，例如 `帧生成 1234ms`。
- 视频加入队列后按钮会变成“已在队列 / 生成中 / 已生成 / 重试视频 VR”。
- 视频 worker 改为进程级任务，进入后台后会继续尽量生成视频；图片预加载仍可随 VR 状态停止。

## v0.11 功能

- 系统图库混排图片和视频；视频缩略图带播放标记。
- 图片自动预加载仍只处理图片，遇到视频会跳过并继续向外补位。
- 图片 VR 页顶部显示当前文件名。
- 视频页使用播放器浏览；按钮为“加入 VR 队列”，不会自动参与图片预加载。
- 视频 VR 使用独立队列，最多 3 个视频任务并行，不占用图片当前图 worker 或后台预加载 worker。
- 视频生成按源 FPS 尽量抽帧，逐帧复用图片 SBS 生成算法，输出 H.264 MP4，并尝试保留原音频轨。
- 生成管理页的视频分页显示视频任务/缓存缩略图，READY 视频可播放、保存到系统相册或删除缓存。
- 调试页对视频显示队列状态、帧进度、FPS、输出路径和错误日志。

## 模型资产

APK 不内置 `depth_anything_v2.tflite`。首次生成 VR 前会从 GitHub Release 下载模型并校验 SHA-256：

```text
https://github.com/7116-byte/ParallelVrGallery/releases/download/model-assets-v1/depth_anything_v2.tflite
```

## 第三方项目

本项目使用 Android / Jetpack Compose、TensorFlow Lite、TFLite GPU Delegate 和 Depth-Anything-V2 模型。详见 `NOTICE.md`。
